# Contribuindo

1. Crie uma branch curta e objetiva.
2. Rode `make test` antes de enviar alterações.
3. Não adicione comandos que alterem recursos OpenShift.
4. Não adicione coleta de conteúdo de Secrets.
5. Atualize documentação, prompts ou runbooks quando mudar comportamento.

Padrões: Bash com `set -Eeuo pipefail`, Python 3.10+, comandos como listas de argumentos, relatórios em Markdown e evidências sanitizadas.
